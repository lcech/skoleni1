TDI

TDI - Turbocharged DOM Infusion. Pragmatic approach to HMTL page DOM updating while page state is still kept on server.

TDI is an AJAX javascript library which:

  • automaticaly processes user actions (clicks, submits, ...)
  • uses a standardized protocol to comunicate with server
  • automaticaly executes server instructions
  • allows to execute multiple server instructions triggered by one user action

Why TDI?

Contrary to the common approach, TDI moves the decision of what to modify in UI from client to server. That means, that the server, after it recieves the AJAX request:

  • processes the request and executes the requested actions (adds a DB record, sends email, ...)
  • in the response sends the user a set of instructions on how to modify the UI (add a new item to the basket, show a user message, execute a script, add a new style, reload the page, ...)

This allows to have one small and uniform piece of javascript, which:

  • processes the user action (click on a link, submission of a form) and sends a request to the server
  • recieves and executes the server instructions to modify the page UI (add a new item to the basket, show a user message, ...)
